# Third-Party Suno Android App Agent Guidelines

## Overview

- You have full freedom being a brand new empty project.
- Commit and push at appropriate points.
- If user needs to install packages,or other places that user interaction is needed, then inform them and they will inform you to continue when done.
- Make classes single intent focused with the codebase as granular as desired: directory structure nesting is FREE, whereas agents reading a 5000 line monolith is NOT.
- Assume massive expansion in the future, i.e. think with long-term project growth and development in mind.
- Use image generation abilities when needed. Benerally assume dark themes are preferred (i.e. a "Chad", "leet", or "I use arch, btw" mindset, of which inserting little nuggets of nerd humour is allowed to give the project class where not obstructing anyting and especailly in the documents which only nerds will read). SVG is also effective even if a placeholder. Modern standards are a good base starting point such as material, or the equivilent of QML with Qt GUI's.
- Don't box yourself into a corner. Ensure refactoring, even heavily, is as painless as possible.
- External libraries or making your own libraries to simplify or eliminate repetative code and to clean up the codebase is usually ideal.
- Have fun and make the project "slap" instead of just being more AI slop.
- You have near-unlimited token allowances including input, output, and thinking.
- You may save information deemed valuable and not obvious or that took time to come to a conclusion in appropriate files.
